# STARBASE

> **"If not now, it will be too late."**

Framework d'exécution personnelle inspiré de la doctrine Starbase — conçu pour 2026.

---

## La mentalité

**Speed wins. Speed dominates.**

Cette mentalité repose sur un constat simple : dans un monde qui accélère exponentiellement, la lenteur est un risque existentiel. Les comités qui "coordonnent mais ne décident jamais", les projets sans owner, les deadlines élastiques — tout ça, c'est terminé.

---

## Principes fondamentaux

| # | Principe | En pratique |
|---|----------|-------------|
| 1 | **Speed > Perfection** | Les risques d'aller trop lentement > les impacts d'un alignement imparfait |
| 2 | **Un seul owner** | Chaque projet = 1 personne accountable, pas un comité |
| 3 | **Deadlines agressifs** | 7 jours pour décider, 30 jours pour livrer |
| 4 | **Fail → Learn** | "We never fail = we never learn" — flip that dynamic |
| 5 | **IA-first** | Pas du "pixie dust" cosmétique, une transformation profonde |
| 6 | **Anti-bureaucratie** | Les bloqueurs = risques opérationnels à éliminer |
| 7 | **Weekly accountability** | "What have you accomplished this week?" |
| 8 | **Data sharing** | Le "data hoarding" est un risque — partager par défaut |
| 9 | **3 outcomes only** | Gamechanging tech, produits scalables, ou nouvelles méthodes |
| 10 | **Never stop iterating** | "The cycle never stops. Always iterating." |

## Formules-clés

Ces phrases ne sont pas des slogans vides — ce sont des **triggers mentaux** pour l'exécution :

- **"Accelerate like hell"** — Quand tu sens que tu ralentis
- **"Work somewhere else"** — Ce que tu dis aux pensées qui bloquent
- **"Preferably with a chainsaw"** — Pour couper dans la bureaucratie mentale
- **"No sacred cows, no exceptions"** — Pas d'excuses, pas de zones protégées
- **"Warp speed"** — Le standard, pas l'exception

## Comment utiliser ce repo

1. **Lis [DOCTRINE.md](./DOCTRINE.md)** — Internalise les principes
2. **Configure tes [METRICS.md](./METRICS.md)** — Définis tes KPIs personnels
3. **Lance un projet** — Utilise le template `pace-project.md`
4. **Weekly review** — Chaque semaine : "Qu'as-tu accompli?"
5. **Iterate** — "The cycle never stops"
